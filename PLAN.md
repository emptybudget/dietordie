# PLAN — 지속가능 다이어트 기록 앱

## 목표

계정 없이 누구나 URL만 열어 바로 쓰는 다이어트 로그.
성공 기준 두 가지: **처음 온 사람이 설명 없이 쓸 수 있다** + **매일 10초 안에 뭐라도 적을 수 있다.**
다이어트의 지속가능성 = 기록의 지속가능성이라는 전제로 설계한다.
데이터는 각 사용자의 기기 브라우저에만 저장된다 — 이것이 계정 없이 대중 공개가 가능한 구조의 핵심이자
프라이버시 강점이므로, UI에서 자산으로 드러낸다.

## 비목표

칼로리 DB · 로그인 · 차트 라이브러리 · 프레임워크/빌드 도구 · 사진 · 푸시 알림 · AI API 직접 호출 ·
온보딩 모달 · 다크모드 · 다국어 (전체 목록은 CLAUDE.md)

## 공개 기준선 — 출시 판정 체크리스트

이 앱이 "대중이 바로 편하게 이용할 수 있다"고 판정하는 기준. 6단계에서 전 항목 검수한다.

**A. 첫 사용**
- [ ] A1. URL을 열고 15초 안에 무슨 앱인지, 지금 뭘 하면 되는지 안다 (빈 화면 안내 문구로)
- [ ] A2. 계정·권한·설정 요구 없이 탭 3번 이내에 첫 기록 완료
- [ ] A3. AI 복붙 흐름(복사→외부 AI→답 저장)이 요약 탭의 화면 내 설명만으로 이해된다

**B. 일상 사용**
- [ ] B1. 재방문 시 오늘 화면이 바로 뜨고, 자주 입력한 항목은 최근 칩으로 원탭 입력된다
- [ ] B2. 터치 타겟 44px 이상, 한 손 조작 가능, 키보드가 올라와도 레이아웃이 깨지지 않는다
- [ ] B3. 홈 화면에 설치하면 오프라인에서도 기록된다

**C. 신뢰·데이터**
- [ ] C1. "데이터는 이 기기에만 저장"이 첫 화면과 설정 탭에 명시된다
- [ ] C2. 백업 내보내기/가져오기가 설정에서 2탭 안에 가능하다
- [ ] C3. 유실 리스크 안내: 브라우저 데이터 삭제·기기 변경 시 사라짐 → 백업 유도.
      iOS Safari 미설치 사용자에게는 홈 화면 추가 안내 1줄(닫기 가능, 닫으면 기억)
- [ ] C4. 항목 삭제 시 5초 실행취소 토스트로 복구 가능하다

**D. 품질**
- [ ] D1. 첫 로드 총 용량 100KB 이하 (dependency 0이므로 자동 충족 예상, 검수에서 확인만)
- [ ] D2. 모든 입력에 label, 아이콘 버튼에 aria-label
- [ ] D3. iOS Safari + Android Chrome 실기기에서 전체 흐름 확인

**E. 문구**
- [ ] E1. 전부 한국어, 기술 용어 노출 없음, 존댓말 통일

## 아키텍처

- **Phase 1 (공개 버전)**: Vercel 정적 배포 + localStorage + PWA. 서버 코드 0줄.
  사용자 간 데이터 공유 없음 — 각자의 브라우저가 각자의 저장소.
- **Phase 2 (운영자 개인 옵션, 공개 배포에 미노출)**: Upstash Redis 무료 티어 + Vercel 함수 1개로
  본인 기기 간 동기화. 공유 토큰 방식은 다수 사용자에게 열 수 없는 구조이므로 공개하지 않는다.
  대중의 기기 이동 수단은 백업 파일(C2)이다.
  **착수 조건**: 본인이 폰·데스크톱 병행 사용으로 실제 불편을 느낄 때. 그 전에는 만들지 않는다.

## 데이터 스키마 (v2)

localStorage 키:

| 키 | 내용 |
|---|---|
| `ddiet:meta` | `{ schemaVersion: 2, iosBannerDismissed?: true, profile?: { heightCm, birthYear } }` |
| `ddiet:day:YYYY-MM-DD` | 하루 기록 (아래 day 객체) |
| `ddiet:summaries` | `[{ id, from, to, text, savedAt }]` — AI 요약 보관 |

day 객체:

```json
{
  "meals":    [{ "id": "...", "time": "12:30", "text": "김치찌개, 밥 반공기" }],
  "workouts": [{ "id": "...", "text": "계단 10층", "minutes": 5, "detail": "3세트×12회" }],
  "sleep":    { "start": "23:40", "end": "07:10" },
  "condition": 4,
  "weight": 70.5,
  "note": "",
  "updatedAt": "2026-07-10T08:12:00+09:00"
}
```

규칙:
- id는 `crypto.randomUUID()`
- 수면은 **기상일** 날짜에 저장. start > end이면 start는 전날 밤으로 해석 (수면시간 계산에 반영)
- sleep·condition·weight는 없으면 null, minutes·detail은 선택 입력
- detail은 자유 텍스트 1칸 (세트·횟수·거리·심박 등 — 구조화된 별도 칸을 늘리지 않는다)
- 체중은 day에 그날그날 기록 (변동 추적은 기록된 날들에서 파생)
- 키·출생연도는 meta.profile에 저장, 설정 탭에서 입력
- 모든 읽기/쓰기는 storage.js 경유, 쓰기 시 updatedAt 갱신
- **최근 칩은 저장하지 않는다**: 최근 7일 day 기록에서 텍스트 빈도순 상위 6개를 파생 계산
  (식사·운동 각각). 별도 키·등록 UI 없음

## 화면 (탭 3개)

문구는 예시이며 톤을 유지하는 선에서 다듬어도 된다.

1. **오늘**
   - 날짜 이동(← 오늘 →), 기록 스트릭(연속 기록일) 표시
   - 식사/운동: 한 줄 입력 + 추가 버튼, 입력창 위에 최근 칩(원탭 입력)
   - 식사 시간은 목록에서 바로 탭해 수정 가능 (추가 시엔 현재 시각 자동)
   - 운동: 분 입력 유지 + 상세 자유 입력 1칸 (세트·거리·심박 등)
   - 각 항목에 × 삭제 → 5초 실행취소 토스트 (토스트 컴포넌트는 1종, 저장 피드백에도 재사용)
   - 수면: 시작/종료 `<input type="time">` · 컨디션: 1~5 버튼 · 체중: kg 입력(선택)
   - 하단 **AI 분석** 카드: 하루 기록을 칼로리 분석 프롬프트로 복사 (아래 사양 참조)
   - 빈 화면 안내(첫 방문): "오늘 먹은 것, 짬짬이 한 운동, 잔 시간을 가볍게 적어보세요.
     기록은 이 기기에만 저장돼요."
2. **요약**
   - 상단 고정 안내: "① 기간을 고르고 프롬프트를 복사하세요 ② ChatGPT, Claude 등
     쓰시는 AI에 붙여넣으세요 ③ 받은 답을 아래에 붙여넣고 저장하세요"
   - 기간 선택(기본 최근 7일) → **프롬프트 복사** 버튼
   - AI 답변 붙여넣는 textarea + 저장 · 저장된 요약 목록
3. **설정**
   - 내 정보: 키(cm)·출생연도 입력 → AI 프롬프트에 포함 (미입력 시 오늘 탭에 안내 1줄)
   - JSON 내보내기/가져오기 (가져오기는 병합이 아닌 확인 후 전체 교체 — 단순하게)
   - 데이터 안내: "기록은 이 기기 브라우저에만 저장됩니다. 브라우저 데이터를 지우거나
     기기를 바꾸면 사라지니, 가끔 내보내기로 백업해 두세요."
   - 앱 한 줄 소개 + 버전

iOS 설치 안내: iOS Safari이고 미설치 상태(`navigator.standalone` false)일 때만
"홈 화면에 추가해서 쓰면 기록이 안전하게 유지돼요 — 공유 버튼 → 홈 화면에 추가" 1줄 배너.
닫으면 meta에 기록하고 다시 띄우지 않는다.
(근거: iOS Safari는 오래 방문하지 않은 사이트의 저장 데이터를 삭제할 수 있음. 설치 시 해당 없음)

## 프롬프트 생성기 사양

프롬프트는 두 종류. 복사는 둘 다 `navigator.clipboard.writeText`,
실패 시 프롬프트를 textarea로 노출해 수동 복사 fallback.

**기간 요약 프롬프트 (요약 탭)** — 세 부분:

1. **데이터**: 내 정보(키·출생연도) + 기간 내 일자별 식사·운동(상세 포함)·수면시간(계산값)·컨디션·체중
2. **분석 지시**: 패턴, 수면↔컨디션↔식사 상관, 체중 변화 흐름, 지속가능성 관점의 조언
3. **답변 형식 지정**: 평문(마크다운 금지), 요약 5문장 이내 + 다음 주 제안 최대 2개
   → 받은 답을 그대로 붙여넣어 저장하기 깔끔하도록

**하루 분석 프롬프트 (오늘 탭)** — 하루치를 다 적은 뒤 복사해 칼로리 분석 받기:

1. **내 정보**: 키·최근 체중·출생연도 (있는 것만)
2. **데이터**: 그날 식사(시간 포함)·운동(상세·분)·수면·컨디션
3. **분석 지시**: 섭취 칼로리 간단 추정 + 운동 소모 칼로리 추정 + 과부족 한두 가지 +
   **마지막 줄에 격려 한마디**
4. **답변 형식**: 평문, 짧게
- 식사·운동이 둘 다 없으면 복사 대신 "먼저 기록해 주세요" 안내

## 빌드 순서 (각 단계 = Claude Code 한 세션 권장)

| # | 작업 | 모델 | 검증 |
|---|---|---|---|
| 1 | storage.js + 스키마 + migrate 뼈대 | opus | 콘솔에서 day CRUD → 새로고침 후 데이터 유지 |
| 2 | 오늘 탭 (3종 기록 + 날짜 이동 + 컨디션 + 삭제/실행취소) | sonnet | 23:40~07:10 수면이 기상일 귀속·시간 계산 정확 · 삭제 후 실행취소로 복원됨 |
| 3 | 요약 탭 (3단계 안내 + 프롬프트 생성·복사 + 요약 저장) | sonnet | 복사한 프롬프트를 실제 AI에 넣어 지정 형식으로 답이 옴 · 안내문만 보고 흐름이 이해되는지 소리 내어 점검 |
| 4 | 최근 칩 + 스트릭 + 백업/복원 | sonnet | 칩이 빈도순 상위로 뜨고 탭 시 즉시 기록됨 · 내보낸 JSON을 시크릿 창에서 복원 → 데이터 동일 |
| 5 | PWA (manifest + network-first SW) + iOS 설치 안내 | haiku~sonnet | 홈 화면 설치·비행기 모드에서 열림·재배포 후 새 버전 반영 · iOS 배너 닫으면 재노출 없음 |
| 6 | **공개 기준선 검수** | sonnet + 사람 | 시크릿 창에서 A~E 전 항목 체크 · 지인 1명에게 URL만 보내 무설명으로 "식사 1개 기록 + AI 요약 저장" 완주 → 막힌 지점 전부 수정 후 재검수 |
| 7 | (P2·비공개) /api/sync + Upstash + 수동 동기화 | opus 설계 → sonnet 구현 | 기기 A 기록 → 동기화 → 기기 B 반영 |

- 모델 전환: Claude Code에서 `/model opus` / `/model sonnet` / `/model haiku`
- 문구·CSS 미세 수정, 재현 확실한 자잘한 버그는 haiku로 처리
- 각 세션 첫 프롬프트: "CLAUDE.md 원칙을 따르고 PLAN.md의 N단계를 진행해" 한 줄이면 충분
- 6단계에서 지인 테스트는 코드로 대체 불가 — 사람이 직접 수행

## Phase 2 상세 (착수 시에만 읽기 · 공개 배포 미노출)

- Vercel 함수 `/api/sync` 하나: GET(전체 데이터 반환), PUT(키 단위 upsert)
- Upstash Redis REST 사용. 키: `day:YYYY-MM-DD`, `summaries`
- 인증: `Authorization: Bearer <SYNC_TOKEN>` ↔ Vercel 환경변수 비교. 계정 시스템 없음
- 클라이언트: URL 파라미터 또는 숨김 진입으로만 접근하는 동기화 설정에 토큰 1회 입력 →
  수동 **동기화** 버튼 = pull → merge(updatedAt 큰 쪽 승리) → push
- 자동/실시간 동기화는 만들지 않는다. 1인 사용이므로 키 단위 last-write-wins로 충분
